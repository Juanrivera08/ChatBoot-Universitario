║         USH CHATBOT — INSTRUCCIONES PARA EL SERVIDOR            ║
╚══════════════════════════════════════════════════════════════════╝

ANTES DE EJECUTAR EL SCRIPT, HAY DOS COSAS QUE DEBES HACER:

┌─────────────────────────────────────────────────────────────────┐
│  PASO 1 — Editar el archivo backend\.env                        │
│                                                                 │
│  Abre ese archivo con el Bloc de notas y cambia:                │
│                                                                 │
│  ALLOWED_ORIGINS=  → el dominio de WordPress                    │
│                      ej: https://ush.edu.co                     │
│                                                                 │
│  DB_HOST=          → IP del servidor PostgreSQL                 │
│  DB_USER=          → usuario de PostgreSQL                      │
│  DB_PASSWORD=      → contraseña de PostgreSQL                   │
│                                                                 │
│  OPENAI_API_KEY=   → clave de OpenAI (GPT)                      │
│                      Obtenerla en:                              │
│                      platform.openai.com/api-keys              │
│                                                                 │
│  El JWT_SECRET YA ESTÁ generado, no lo cambies.                 │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  PASO 2 — Ejecutar el script de instalación                     │
│                                                                 │
│  Archivo: 2-instalar-en-servidor.ps1                            │
│                                                                 │
│  → Clic derecho sobre ese archivo                               │
│  → "Ejecutar como administrador"                                │
│                                                                 │
│  El script hace todo solo:                                      │
│    ✓ Instala Node.js si no está                                 │
│    ✓ Instala el gestor de procesos (PM2)                        │
│    ✓ Pone el backend a correr en el puerto 3001                 │
│    ✓ Publica el widget en IIS                                   │
│    ✓ Al final muestra las URLs y el código para WordPress       │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  PASO 3 — Agregar el widget a WordPress                         │
│                                                                 │
│  El script del paso 2 te dará el código exacto al final.        │
│  Será algo así (con la IP real del servidor):                   │
│                                                                 │
│  1. En WordPress, instalar el plugin gratuito "WPCode"          │
│     (buscar en Plugins → Añadir nuevo)                          │
│                                                                 │
│  2. Ir a WPCode → Header & Footer → pestaña "Footer"           │
│                                                                 │
│  3. Pegar el código que muestra el script                       │
│                                                                 │
│  4. Guardar. El botón del chat aparece en todo el sitio. ✓     │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  ⚠ IMPORTANTE — SI EL SERVIDOR YA TENÍA EL CHATBOT CON GEMINI    │
│                                                                 │
│  Cambiamos la IA de Google Gemini a OpenAI (GPT). Esto obliga   │
│  a dos pasos extra UNA SOLA VEZ, después de copiar esta carpeta │
│  y antes (o justo después) de reiniciar el backend:             │
│                                                                 │
│  1) Actualizar el modelo guardado en la base de datos:          │
│       psql -U <usuario> -d ush_chatbot -f backend\sql\          │
│            migrate_to_openai.sql                                 │
│                                                                 │
│  2) Re-indexar los documentos con los embeddings de OpenAI      │
│     (los vectores de Gemini son incompatibles). Desde la        │
│     carpeta backend:                                            │
│       cd backend                                                │
│       node dist\scripts\reindexAll.js                           │
│                                                                 │
│  Sin estos dos pasos, el chatbot NO encontrará información en   │
│  los documentos. En una instalación NUEVA no hacen falta.       │
└─────────────────────────────────────────────────────────────────┘

  pm2 status          → ver si el backend está corriendo
  pm2 logs            → ver errores en tiempo real
  pm2 restart all     → reiniciar si algo falla
  pm2 restart all --update-env → recargar tras cambiar el .env
